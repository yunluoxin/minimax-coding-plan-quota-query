# MiniMaxMenuBar 产品需求文档

## 1. 产品概述

**产品名称**: MiniMaxMenuBar
**产品类型**: macOS 菜单栏工具应用
**一句话描述**: 在 macOS 状态栏实时显示 MiniMax API 剩余配额百分比
**目标用户**: 使用 MiniMax API 的开发者

## 2. 背景与动机

MiniMax API 有使用配额限制，用户需要频繁访问网页或调用接口才能查看剩余配额。本工具将配额信息直接展示在 macOS 菜单栏，让用户随时了解配额使用情况。

## 3. 用户故事

| 角色 | 故事 |
|------|------|
| 开发者 | 作为开发者，我希望在菜单栏看到 API 剩余配额，以便知道还能调用多少次 |
| 开发者 | 作为开发者，我希望在配额快用完时得到视觉提醒，以便及时规划 |
| 开发者 | 作为开发者，我希望配置一次后长期使用，无需反复输入凭证 |
| 开发者 | 作为开发者，我希望快速刷新查看最新数据，以便测试时即时确认 |

## 4. 功能需求

### 4.1 核心功能

#### F1: 状态栏显示
- 在 macOS 菜单栏显示剩余配额百分比
- 未配置时显示 `N/A`
- 请求失败时显示 `⚠️`
- 正常时显示 `XX%` 格式

#### F2: 配额详情弹窗
- 点击状态栏图标打开 Popover 弹窗
- 显示当前模型名称
- 显示配额时间窗口 (开始时间、结束时间)
- 显示剩余时间倒计时
- 显示已用配额 / 总配额
- 颜色指示剩余比例 (绿/黄/红)

#### F3: 配置管理
- 首次使用时弹出设置界面
- 输入 API Key (必填)
- 输入 Group ID (可选)
- 保存配置到本地 (UserDefaults)
- 支持修改和清除配置

#### F4: 数据刷新
- 启动时自动刷新
- 每 5 分钟自动刷新
- 手动点击刷新按钮
- 打开弹窗时刷新

#### F5: 退出应用
- 提供退出按钮
- 点击即可完全退出应用

#### F6: 每日/每周使用统计
- 记录每天剩余配额快照到本地
- 显示本周已用/总配额（API 直接计算）
- 显示近7天每日使用量柱状图
- 显示日均和最大使用量
- 数据持久化，保留30天
- 存储大小 < 5MB，自动清理旧数据

### 4.3 错误处理

| 场景 | 状态栏 | 弹窗内容 |
|------|--------|----------|
| 未配置 | `N/A` | 提示配置 API Key |
| 网络错误 | `⚠️` | 显示错误信息 |
| API 返回错误 | `⚠️` | 显示 API 错误码和消息 |
| 解析失败 | `⚠️` | 提示解析错误 |
| 未知错误 | `⚠️` | 显示错误描述 |

## 5. 非功能需求

### 5.1 性能
- 启动时间 < 2 秒
- API 请求超时 15 秒
- 自动刷新间隔 300 秒
- 内存占用 < 50MB

### 5.2 兼容性
- 最低 macOS 14.0
- 支持 Apple Silicon 和 Intel

### 5.3 隐私与安全
- API Key 仅存储在本地 UserDefaults
- 不上传任何用户数据
- 不收集任何隐私信息

### 5.4 可用性
- 无需签名即可本地调试
- LSUIElement = true，不占用 Dock 图标
- 透明弹窗，点击外部自动关闭

## 6. 技术约束

- **框架**: SwiftUI (UI) + AppKit (状态栏)
- **语言**: Swift 5.9+
- **构建工具**: XcodeGen
- **依赖**: 无第三方依赖 (纯 Apple 框架)
- **权限**: 网络权限 (查询 API)
